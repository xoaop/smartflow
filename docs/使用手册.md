# SmartFlow 团队效能周报系统使用手册

## 系统简介
SmartFlow 是一个基于飞书OpenClaw的自动化团队效能周报生成系统，能够自动采集飞书文档、任务、会议数据，通过Claude大模型智能提炼生成高信息密度的周报，定时推送到飞书群，大幅提升团队信息流转效率。

## 功能特性
- 🚀 **多源数据采集**：自动拉取飞书文档、任务、会议多源数据
- 🤖 **智能内容生成**：基于Claude大模型自动提炼总结，生成专业周报
- 📅 **多周期支持**：支持周/双周/月度等多种生成周期
- 👥 **多团队隔离**：支持多个团队独立配置，数据完全隔离
- ⏰ **定时自动推送**：灵活的定时任务配置，到点自动推送飞书卡片
- ⚠️ **智能风险识别**：自动识别任务延期、决议未跟进等潜在风险
- 🔗 **来源溯源**：所有内容都标注原始来源链接，支持点击跳转

## 环境要求
- Node.js >= 20.0.0
- 飞书企业自建应用权限
- Claude API Key

## 快速开始

### 1. 安装依赖
```bash
npm install
```

### 2. 编译TypeScript
```bash
npm run build
```

### 3. 全局安装CLI
```bash
npm link
```

### 4. 配置飞书应用
1. 登录[飞书开放平台](https://open.feishu.cn/)，创建企业自建应用
2. 申请以下权限：
   - 文档：`docx:document:readonly`, `drive:drive:readonly`
   - 任务：`task:task:readonly`, `task:tasklist:readonly`
   - 日历：`calendar:calendar:readonly`
   - 会议：`vc:meeting:readonly`, `minutes:minutes:readonly`
   - 消息：`im:message:send`, `im:chat:readonly`
3. 发布应用并申请企业授权

### 5. 配置系统
```bash
# 首次运行会自动创建配置目录
smartflow health
```

### 6. 编辑全局配置
复制配置模板到 `~/.smartflow/config/config.yaml`：
```bash
cp config/config.example.yaml ~/.smartflow/config/config.yaml
```

编辑配置文件，填写Claude API Key等信息。

### 7. 创建团队配置
```bash
# 创建团队配置模板
smartflow config create my-team "我的团队"
```

编辑生成的配置文件 `~/.smartflow/config/teams/my-team.yaml`，填写飞书AppId、AppSecret、数据源配置等信息。

### 8. 测试配置
```bash
# 测试推送
smartflow push test my-team
```

### 9. 手动生成周报
```bash
# 生成上周周报并推送到配置的渠道
smartflow generate run --team my-team --range lastweek --push
```

### 10. 启动定时任务
```bash
# 启动定时任务服务，会自动按照配置的cron表达式生成并推送周报
smartflow schedule start
```

## 命令说明

### 配置管理
```bash
# 列出所有团队配置
smartflow config list

# 创建新的团队配置模板
smartflow config create <teamId> <teamName>

# 显示指定团队的配置
smartflow config show <teamId>

# 删除指定团队的配置
smartflow config delete <teamId>
```

### 周报生成
```bash
# 手动生成周报
smartflow generate run --team <teamId> [选项]
选项：
  -r, --range <range>  时间范围: lastweek, thisweek, lastmonth, thismonth, 或自定义如 2024-01-01~2024-01-07
  -p, --push           生成后自动推送
  -o, --output <file>  输出到JSON文件

# 仅采集数据，不生成报告
smartflow generate collect --team <teamId> --range lastweek --output data.json
```

### 推送功能
```bash
# 测试推送功能
smartflow push test <teamId>

# 推送本地JSON文件中的周报
smartflow push file <teamId> <filePath>
```

### 定时任务管理
```bash
# 启动定时任务服务
smartflow schedule start

# 列出所有定时任务
smartflow schedule list

# 手动触发指定团队的任务
smartflow schedule trigger <teamId>

# 为团队添加定时任务
smartflow schedule add <teamId>

# 移除团队的定时任务
smartflow schedule remove <teamId>
```

### 其他命令
```bash
# 健康检查
smartflow health

# 查看版本
smartflow --version

# 查看帮助
smartflow --help
```

## Cron表达式说明
定时任务使用标准cron表达式，格式为：`秒 分 时 日 月 周`（本系统默认秒为0，可以省略）

常用示例：
- `0 18 * * 5`：每周五18:00
- `0 9 * * 1`：每周一09:00
- `0 10 1 * *`：每月1号10:00
- `0 18 */14 * *`：每14天18:00（双周）

## 飞书权限配置说明

### 文档权限
- 要采集的文档目录需要给飞书应用开通查看权限
- rootFolderToken可以从飞书文档URL中获取：https://example.feishu.cn/drive/folder/[folderToken]

### 任务权限
- 项目ID可以从飞书任务项目的URL中获取：https://example.feishu.cn/task/project/[projectId]

### 日历权限
- 日历ID可以通过飞书开放平台接口获取，或从日历设置中查看

### 群聊ID
- 群聊ID可以通过飞书开放平台接口获取，或在群设置中查看

## 常见问题

### Q: 飞书API调用提示权限不足？
A: 请检查飞书应用是否申请了对应的权限，并且已经发布和获得企业授权。同时确认应用是否有对应资源（文档目录、任务项目、日历）的访问权限。

### Q: 生成的周报内容不准确或有幻觉？
A: 1. 确保飞书内的原始数据完整且规范；2. 调整提示词模板，增加更多约束；3. 在配置中提高detailLevel级别。

### Q: 定时任务不触发？
A: 1. 检查cron表达式是否正确；2. 确认定时任务服务正在运行；3. 查看日志文件 `~/.smartflow/logs/` 排查错误。

### Q: 如何支持多个团队？
A: 每个团队创建独立的飞书应用和配置文件，系统会自动隔离数据。多个团队的定时任务可以在同一个服务中运行。

```
src/
├── cli/                  # CLI命令定义
├── modules/              # 业务模块
│   ├── config/          # 配置管理
│   ├── collector/       # 数据采集
│   ├── generator/       # 内容生成
│   ├── push/            # 推送分发
│   └── schedule/        # 定时任务
├── common/               # 通用工具
│   ├── feishu/          # 飞书API封装
│   ├── llm/             # 大模型封装
│   ├── db/              # 数据库操作
│   ├── logger/          # 日志工具
│   └── utils/           # 工具函数
└── types/                # TypeScript类型定义
```

## 数据存储
所有数据存储在 `~/.smartflow/` 目录下：
- `config/`：配置文件目录
- `data/`：SQLite数据库文件
- `logs/`：日志文件
