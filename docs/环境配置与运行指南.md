# 环境配置与运行指南

## 一、运行环境要求
### 1.1 基础环境
| 依赖 | 最低版本 | 说明 |
|-----|---------|------|
| Node.js | 20.0.0 | 建议使用LTS版本，支持ESModule和Top-Level Await |
| npm/yarn/pnpm | 任意最新版本 | 包管理工具 |
| 操作系统 | Windows 10+/macOS 10.15+/Linux | 跨平台支持 |

### 1.2 第三方依赖
- **飞书企业自建应用**：需要在飞书开放平台创建并配置相应权限
- **Claude API Key**：用于大模型内容生成，可在Anthropic官网申请
- 网络要求：能够访问飞书开放平台和Claude API（如有需要可配置代理）

## 二、开发环境搭建
### 2.1 源码下载
```bash
git clone <仓库地址>
cd smartflow
```

### 2.2 安装依赖
```bash
# 使用npm
npm install

# 或使用yarn
yarn install

# 或使用pnpm
pnpm install
```

### 2.3 TypeScript编译
```bash
# 开发模式（监听文件变化自动编译）
npm run dev

# 生产模式（一次性编译）
npm run build
```

### 2.4 全局链接（方便本地测试）
```bash
npm link
```
现在你可以在任意位置使用 `smartflow` 命令。

### 2.5 环境验证
```bash
# 查看版本
smartflow --version

# 健康检查
smartflow health
```
如果看到"✅ 服务运行正常"的提示，说明环境配置成功。

## 三、配置文件说明
首次运行会自动在用户目录下创建 `.smartflow` 目录：
```
~/.smartflow/
├── config/              # 配置文件目录
│   ├── config.yaml      # 全局配置文件
│   └── teams/           # 团队配置目录
│       └── *.yaml       # 各个团队的配置文件
├── data/                # 数据存储目录
│   └── smartflow.db     # SQLite数据库文件
└── logs/                # 日志目录
    ├── error.log        # 错误日志
    └── combined.log     # 全量日志
```

### 3.1 全局配置（config.yaml）
```yaml
# 默认团队ID
defaultTeamId: "demo-team"

# 日志级别: error/warn/info/debug
logLevel: "info"

# 数据存储目录
dataDir: "~/.smartflow/data"

# 大模型配置
llm:
  apiKey: "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"  # Claude API Key
  baseUrl: "https://api.anthropic.com"           # API地址（支持代理）
  model: "claude-3-5-sonnet-20240620"            # 使用的模型
```

### 3.2 团队配置（teams/[teamId].yaml）
参考 `config/team-config.example.yaml` 进行配置，每个团队独立配置。

## 四、运行方式
### 4.1 手动运行（适合测试）
```bash
# 生成上周周报并推送
smartflow generate run --team my-team --range lastweek --push

# 生成本周周报但不推送，输出到文件
smartflow generate run --team my-team --range thisweek --output ./report.json

# 仅采集数据，不生成报告
smartflow generate collect --team my-team --range lastweek --output ./data.json
```

### 4.2 定时任务模式（适合生产环境）
```bash
# 启动定时任务服务
smartflow schedule start

# 查看定时任务列表
smartflow schedule list

# 手动触发某个团队的任务
smartflow schedule trigger my-team
```

#### 后台运行（Linux/macOS）
```bash
# 使用nohup后台运行
nohup smartflow schedule start > /dev/null 2>&1 &

# 或使用pm2管理进程
npm install -g pm2
pm2 start "smartflow schedule start" --name smartflow
pm2 save
pm2 startup
```

#### Windows后台运行
```powershell
# 使用PowerShell后台运行
Start-Process -NoNewWindow -FilePath "smartflow" -ArgumentList "schedule", "start"
```

### 4.3 Docker运行
创建Dockerfile：
```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build
RUN npm link

# 挂载配置和数据目录
VOLUME ["/root/.smartflow"]

CMD ["smartflow", "schedule", "start"]
```

构建运行：
```bash
docker build -t smartflow .
docker run -d \
  -v ~/.smartflow:/root/.smartflow \
  --name smartflow \
  smartflow
```

### 4.4 飞书OpenClaw运行
参考《飞书OpenClaw接入指南》配置后，通过OpenClaw运行：
```bash
# 生成周报
claw smartflow generate run --team my-team --range lastweek

# 启动定时任务
claw smartflow schedule start
```

## 五、飞书应用配置指南
### 5.1 创建飞书应用
1. 访问[飞书开放平台](https://open.feishu.cn/)，创建企业自建应用
2. 填写应用基本信息，上传应用图标
3. 在"权限管理"中申请以下权限：
   | 权限名称 | 权限描述 | 是否必须 |
   |---------|---------|---------|
   | `docx:document:readonly` | 查看文档 | 是（文档采集需要） |
   | `drive:drive:readonly` | 查看云文档文件 | 是 |
   | `task:task:readonly` | 查看任务 | 是（任务采集需要） |
   | `task:tasklist:readonly` | 查看任务清单 | 是 |
   | `calendar:calendar:readonly` | 查看日历 | 是（会议采集需要） |
   | `vc:meeting:readonly` | 查看会议信息 | 是 |
   | `minutes:minutes:readonly` | 查看会议纪要 | 是 |
   | `im:message:send` | 发送消息 | 是（推送需要） |
   | `im:chat:readonly` | 获取群组信息 | 是 |

4. 在"版本管理与发布"中创建版本，申请企业发布
5. 企业管理员审核通过后，应用即可使用

### 5.2 获取必要的ID
- **AppId/AppSecret**：在飞书开放平台应用详情页的"凭证与基础信息"中获取
- **文档根目录Token**：打开飞书文档目录，URL中的`folder/`后面的字符串
- **任务项目ID**：打开飞书任务项目，URL中的`project/`后面的字符串
- **日历ID**：通过飞书API获取，或在日历设置中查看
- **群聊ID**：在飞书群设置->群信息->群ID中获取
- **用户ID**：在飞书通讯录中查看用户的用户ID

### 5.3 权限测试
配置完成后，运行测试命令验证权限：
```bash
# 测试推送功能
smartflow push test my-team

# 测试数据采集
smartflow generate collect --team my-team --range lastweek
```

## 六、常见问题排查
### 6.1 安装依赖失败
```bash
# 清理缓存重新安装
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### 6.2 飞书API调用失败
1. 检查AppId和AppSecret是否正确
2. 确认应用已经发布并获得企业授权
3. 确认应用已经申请了对应的权限
4. 检查网络是否能够访问飞书开放平台
5. 查看日志文件 `~/.smartflow/logs/error.log` 获取详细错误信息

### 6.3 Claude API调用失败
1. 检查API Key是否正确
2. 确认账户有足够的余额
3. 检查网络是否能够访问Claude API（可配置代理）
4. 确认模型名称正确
5. 查看日志文件中的详细错误信息

### 6.4 定时任务不触发
1. 检查cron表达式是否正确（注意是5位还是6位格式）
2. 确认定时任务服务正在运行
3. 检查团队配置中`push.enabled`是否为true
4. 查看执行日志是否有错误信息

### 6.5 生成的周报内容为空
1. 确认时间范围内确实有数据产生
2. 检查数据源配置是否正确（目录Token、项目ID等）
3. 确认飞书应用有对应资源的访问权限
4. 检查过滤规则是否过滤掉了所有内容

## 七、性能优化建议
### 7.1 大团队部署
- 对于团队数量较多的情况，建议分离采集和生成服务
- 使用消息队列异步处理任务，避免阻塞
- 配置数据库索引，提升查询性能

### 7.2 大模型优化
- 对于数据量较大的团队，可以分批处理数据
- 优化提示词，减少不必要的Token消耗
- 使用Prompt缓存，降低API调用成本

### 7.3 高可用部署
- 使用多实例部署，避免单点故障
- 配置监控告警，及时发现异常
- 定期备份数据库和配置文件
