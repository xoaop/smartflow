## 二、核心模块代码讲解

### 2.1 配置管理模块
**核心文件**：`src/modules/config/team-config.service.ts`

#### 设计思路
- 采用单例模式，全局只有一个配置服务实例
- 配置文件使用YAML格式，易于阅读和修改
- 使用Zod进行配置校验，确保配置合法性
- 支持多团队配置独立存储和隔离

#### 关键实现
```typescript
// 单例模式实现
public static getInstance(): TeamConfigService {
  if (!TeamConfigService.instance) {
    TeamConfigService.instance = new TeamConfigService();
  }
  return TeamConfigService.instance;
}

// 配置加载和校验
public async getTeamConfig(teamId: string): Promise<TeamConfig> {
  // 先查内存缓存
  if (this.teamConfigs.has(teamId)) {
    return this.teamConfigs.get(teamId)!;
  }

  // 从文件加载
  const configPath = path.join(this.teamsConfigDir, `${teamId}.yaml`);
  const content = await fs.readFile(configPath, 'utf-8');
  const config = yaml.parse(content);
  
  // Zod校验
  const validatedConfig = TeamConfigSchema.parse(config);
  this.teamConfigs.set(teamId, validatedConfig);
  
  return validatedConfig;
}
```

#### 设计亮点
- 内存缓存+文件存储的二级缓存机制，提升配置加载速度
- 自动创建配置目录和默认配置文件，降低使用门槛
- 完整的配置校验，提前发现配置错误，避免运行时异常

### 2.2 飞书API封装
**核心文件**：`src/common/feishu/client.ts`

#### 设计思路
- 封装飞书SDK的复杂细节，提供统一的调用接口
- 自动管理tenant_access_token的刷新，无需业务层关心
- 实现指数退避重试机制，应对API限流和临时故障
- 支持团队级别的客户端隔离，避免权限混淆

#### 关键实现
```typescript
// Token自动刷新机制
public async getTenantAccessToken(): Promise<string> {
  const now = Date.now();
  // 提前1分钟刷新token
  if (this.tenantAccessToken && now < this.tokenExpireTime - 60 * 1000) {
    return this.tenantAccessToken;
  }

  // 调用API获取新token
  const response = await this.client.auth.tenantAccessToken.internal({
    data: {
      app_id: this.teamConfig.feishu.appId,
      app_secret: this.teamConfig.feishu.appSecret,
    },
  });

  this.tenantAccessToken = response.data.tenant_access_token;
  this.tokenExpireTime = now + response.data.expire * 1000;
  return this.tenantAccessToken;
}

// 通用请求方法，带重试机制
public async request<T = any>(method: string, path: string, options?: any): Promise<T> {
  const retryTimes = options?.retryTimes || 3;
  let lastError: Error | null = null;

  for (let i = 0; i < retryTimes; i++) {
    try {
      const token = await this.getTenantAccessToken();
      const response = await this.client.request({
        method,
        url: path,
        headers: { Authorization: `Bearer ${token}` },
        ...options,
      });
      return response.data as T;
    } catch (error) {
      lastError = error as Error;
      // Token过期自动重试
      if (error.code === 99991663 || error.code === 99991664) {
        this.tenantAccessToken = null;
      }
      // 指数退避
      await new Promise(resolve => setTimeout(resolve, 1000 * Math.pow(2, i)));
    }
  }

  throw lastError;
}
```

#### 设计亮点
- 自动token管理，业务层无需关心认证细节
- 智能重试机制，大幅提升API调用成功率
- 统一的错误处理，简化业务层代码

### 2.3 数据采集模块
**核心文件**：`src/modules/collector/feishu-collector.service.ts`

#### 设计思路
- 采用接口抽象，所有采集器实现统一的`ISourceCollector`接口
- 并行采集多源数据，提升采集效率
- 错误隔离，单个数据源采集失败不影响整体流程
- 可扩展设计，方便后续添加新的数据源

#### 关键实现
```typescript
// 并行采集三类数据
async collect(teamConfig: TeamConfig, timeRange: TimeRange): Promise<CollectedData> {
  // Promise.all并行执行，提升速度
  const [docs, tasks, meetings] = await Promise.all([
    this.collectDocs(teamConfig, timeRange),
    this.collectTasks(teamConfig, timeRange),
    this.collectMeetings(teamConfig, timeRange),
  ]);

  return {
    teamId: teamConfig.teamId,
    timeRange,
    collectedAt: new Date(),
    docs,
    tasks,
    meetings,
  };
}

// 单个数据源采集，错误隔离
private async collectDocs(teamConfig: TeamConfig, timeRange: TimeRange) {
  try {
    const docCollector = new DocCollector(teamConfig);
    return await docCollector.collect(teamConfig, timeRange);
  } catch (error) {
    logger.error('文档采集失败，跳过该数据源', { error });
    return []; // 返回空数组，不影响整体流程
  }
}
```

#### 设计亮点
- 接口驱动设计，易于扩展新的数据源
- 并行采集，充分利用IO等待时间
- 错误隔离，提升系统鲁棒性

### 2.4 内容生成模块
**核心文件**：`src/modules/generator/report-generator.service.ts`

#### 设计思路
- 提示词模板化，便于优化和调整输出格式
- 结构化输出要求，确保大模型返回可解析的JSON格式
- 幻觉校验机制，确保生成内容的真实性
- 来源溯源，所有内容都能找到原始数据支撑

#### 关键实现
```typescript
async generate(collectedData: CollectedData, teamConfig: TeamConfig): Promise<WeeklyReport> {
  // 1. 数据预处理
  const processedData = this.preprocessData(collectedData, teamConfig);
  
  // 2. 构建提示词
  const { systemPrompt, userPrompt } = this.buildPrompts(processedData, teamConfig);
  
  // 3. 调用大模型，要求返回JSON格式
  const reportContent = await this.claudeClient.generateJson<WeeklyReport['content']>(
    userPrompt,
    systemPrompt
  );
  
  // 4. 校验生成内容，防止幻觉
  this.validateReportContent(reportContent, collectedData);
  
  // 5. 补充来源信息
  const sources = this.extractSources(collectedData);

  return {
    teamId: collectedData.teamId,
    timeRange: collectedData.timeRange,
    generatedAt: new Date(),
    content: reportContent,
    sources,
  };
}

// 幻觉校验
private validateReportContent(content: WeeklyReport['content'], collectedData: CollectedData) {
  // 收集所有原始数据的URL
  const allUrls = new Set<string>();
  collectedData.docs.forEach(doc => allUrls.add(doc.url));
  collectedData.tasks.forEach(task => allUrls.add(task.url));
  collectedData.meetings.forEach(meeting => allUrls.add(meeting.url));

  // 检查每个生成内容的来源URL是否存在于原始数据中
  content.keyWork.forEach(item => {
    if (item.sourceUrl && !allUrls.has(item.sourceUrl)) {
      logger.warn('发现可能的幻觉内容', { url: item.sourceUrl, title: item.title });
    }
  });
}
```

#### 设计亮点
- 结构化输出，避免大模型返回不可解析的内容
- 幻觉校验机制，提升内容准确性
- 提示词模板化，便于优化和适配不同场景

### 2.6 定时任务模块
**核心文件**：`src/modules/schedule/scheduler.service.ts`

#### 设计思路
- 基于`node-schedule`实现cron表达式调度
- 任务状态持久化，服务重启自动恢复
- 执行日志完整记录，便于问题排查
- 错误自动重试和告警，提升系统可靠性

#### 关键实现
```typescript
// 调度任务
async scheduleJob(teamConfig: TeamConfig): Promise<void> {
  // 取消已存在的任务
  if (this.jobs.has(teamConfig.teamId)) {
    this.cancelJob(teamConfig.teamId);
  }

  // 创建任务
  const job = schedule.scheduleJob(
    teamConfig.teamId,
    teamConfig.push.cronExpression,
    async () => {
      await this.executeJob(teamConfig); // 执行任务逻辑
    }
  );

  // 持久化到数据库
  this.upsertScheduledTask(teamConfig);
}

// 任务执行逻辑
private async executeJob(teamConfig: TeamConfig): Promise<void> {
  const logId = this.startExecutionLog(teamConfig.teamId, 'weekly_report');

  try {
    // 完整的执行流程
    const timeRange = this.collectorService.parseTimeRange('lastweek');
    const collectedData = await this.collectorService.collect(teamConfig, timeRange);
    const report = await this.generatorService.generate(collectedData, teamConfig);
    const reportId = this.saveReport(report);
    const pushResult = await this.pushService.pushWeeklyReport(report);
    
    this.finishExecutionLog(logId, 'success');
  } catch (error) {
    this.finishExecutionLog(logId, 'failed', error.message);
    // 错误告警
    await this.pushService.pushErrorNotification(error);
  }
}
```

#### 设计亮点
- 任务持久化，重启不丢失
- 完整的执行链路追踪，便于排查问题
- 自动错误告警，及时发现异常

## 三、关键技术点讲解
### 3.1 TypeScript类型系统
项目全程使用TypeScript，所有接口和数据结构都有明确的类型定义：
- 类型定义集中在`src/types/index.ts`
- 配置校验使用Zod实现运行时类型校验
- 编译时类型检查+运行时校验的双重保障

### 3.2 异步并发控制
大量使用`Promise.all`实现IO密集型任务的并行处理：
- 多数据源并行采集
- 多渠道并行推送
- 显著提升系统吞吐量和响应速度

### 3.3 错误处理机制
- 分层错误处理：底层抛出错误，上层处理并决定是否重试
- 错误日志完整记录，包含上下文信息
- 关键流程错误自动告警，及时通知管理员

### 3.4 可扩展性设计
- 所有核心模块基于接口实现，便于替换
- 配置驱动的业务逻辑，无需修改代码即可适配不同场景
- 预留扩展点，方便后续添加新功能
